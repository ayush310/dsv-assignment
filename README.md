# User Management CRUD Application

A React-based user management system with schema-driven architecture for easy extensibility.

## Setup Instructions

1. **Install dependencies**

   ```bash
   npm install
   ```

2. **Start the mock server** (Terminal 1)

   ```bash
   npm run server
   ```

3. **Start the React app** (Terminal 2)

   ```bash
   npm run dev
   ```

4. **Access the application**
   - Open `http://localhost:5173` in your browser

## Adding New Fields

The application uses a **schema-driven architecture** - add fields without touching UI code.

**Steps:**

1. Open `src/config/userSchema.js`
2. Add a new field object to the array:

```javascript
{
  name: 'dateOfBirth',
  label: 'Date of Birth',
  type: 'date',
  required: false,
  placeholder: 'Select date',
  validation: {
    errorMessage: 'Please enter a valid date'
  }
}
```

3. Save - the field automatically appears in forms and validation!

**Supported field types:** text, email, tel, date, number, url, password

**Validation options:**

- `required`: boolean
- `minLength`: number
- `maxLength`: number
- `pattern`: RegExp
- `errorMessage`: string

## Design Decisions

### Architecture

- **Schema-driven forms**: Single configuration file (`userSchema.js`) defines all fields, validation, and UI rendering
- **Service layer**: API calls centralized in `userService.js` for easy backend switching
- **Component composition**: Reusable `FormInput` component renders any field type from schema

### Technology Choices

- **React 18** with functional components and hooks
- **Axios** for HTTP requests with interceptor support
- **JSON Server** for mock API (easy to replace with real backend)
- **CSS Modules approach** with scoped component styles

### Key Features

- Real-time validation with field-level error messages
- Optimistic UI updates with error rollback
- Modal-based forms for better UX
- Toast notifications for user feedback
- Responsive grid layout for user cards

### Assumptions

- Mock API uses JSON Server for development
- User IDs are auto-generated by the backend
- All fields stored as strings (convert as needed for your backend)
- CORS enabled for local development

### File Structure

```
src/
├── components/       # UI components
├── config/          # userSchema.js - field configuration
├── services/        # API client and user service
└── App.jsx          # Main application logic
```

## API Endpoints

The application expects these endpoints:

- `GET /users` - List all users
- `POST /users` - Create user
- `PUT /users/:id` - Update user
- `DELETE /users/:id` - Delete user

To use a different backend, update `baseURL` in `src/services/apiClient.js`.

## Development Notes
